# tg-benz

Telegram-бот для получения актуальной информации о наличии топлива на АЗС с сайта [gdebenz.ru](https://gdebenz.ru).

## Возможности

- Ввод города текстом
- Выбор режима: все АЗС или только те, где есть топливо
- Кнопка «🔄 Обновить» и переключение режима под результатом
- Клик по адресу открывает маршрут в Яндекс Навигаторе
- Кэш запросов (3 минуты) — снижает нагрузку на gdebenz.ru и Nominatim
- Статистика и роли администраторов

## Установка

```bash
git clone <repo>
cd tg-benz
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## Запуск

```bash
cp .env.example .env
# впишите BOT_TOKEN и (опционально) ADMIN_IDS в .env
export $(cat .env | xargs)
python3 bot.py
```

Подробная инструкция по установке на сервер — см. [INSTALL.md](INSTALL.md).

## Конфигурация

| Переменная   | Описание |
|---|---|
| `BOT_TOKEN`  | Токен Telegram-бота (обязательно) |
| `ADMIN_IDS`  | Telegram ID начальных администраторов через запятую: `123456,789012` |

## Команды бота

### Доступны всем пользователям

| Команда | Описание |
|---|---|
| `/start` | Приветствие, кнопка повтора последнего запрошенного города |
| `/help` | Справка |
| `/fuel <город>` | Все АЗС города (для групповых чатов) |
| `/fuelnow <город>` | Только АЗС с топливом (для групповых чатов) |

В личных сообщениях можно просто написать название города —
бот предложит выбрать режим показа кнопками.

### Только для администраторов

Эти команды **не отображаются** в общем меню обычных пользователей — Telegram
показывает их только тем, у кого роль `admin` в базе (через персональный
`BotCommandScopeChat`).

| Команда | Описание |
|---|---|
| `/stats` | Статистика: пользователи, запросы, топ городов, последняя активность |
| `/admins` | Список текущих администраторов |
| `/addadmin <id\|@username>` | Назначить администратора |
| `/removeadmin <id\|@username>` | Снять права администратора |
| `/broadcast <текст>` | Разослать сообщение всем пользователям бота |
| `/export` | Выгрузить статистику пользователей в CSV |
| `/rawstation <город>` | Отладка: сырой JSON-ответ API для первой станции города |

Первый администратор задаётся через `ADMIN_IDS` в `.env`. Чтобы назначить
нового — пользователь должен сначала написать боту `/start` (попасть в базу),
затем текущий админ выполняет `/addadmin`.

## Структура

```
bot.py          — Telegram-бот (aiogram 3.x): хендлеры, клавиатуры, рассылка
scraper.py      — клиент API gdebenz.ru + геокодирование (Nominatim) + кэш
db.py           — SQLite: пользователи, роли, запросы
requirements.txt
.env.example
INSTALL.md      — инструкция по установке на чистый Debian-сервер
```

## Как это работает

1. **Город → координаты**: Nominatim (`nominatim.openstreetmap.org`) геокодирует
   название города.
2. **Координаты → АЗС**: прямой HTTP-запрос к `gdebenz.ru/api/nearby` возвращает
   список станций со статусом (`есть` / `очередь` / `мало` / `нет`).
3. Ответ кэшируется на 3 минуты на ключ «город + радиус», чтобы повторные
   запросы не дёргали внешние API.
